# USG Signal Model (Canonical)

Simple operating chain:

1. Collect signals
2. Identify system
3. Map actors
4. Map lifecycle stage
5. Recognize patterns
6. Update model
7. Predict and decide

## Signals schema fields (Phase 1)
- model_step
- system_id / system_guess
- actor_refs
- actor_roles
- lifecycle_stage_refs
- evidence_type
- evidence_strength
- advances_hypothesis
- blocks_hypothesis

## Platform Pressure fields (Phase 1)
- pattern_summary
- model_update
- recommended_motion
- decision_confidence
- missing_actor_roles
- required_next_evidence
