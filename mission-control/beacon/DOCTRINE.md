# Sponsor-Level Presence Beacon Doctrine (TS-SBP1.1)

## Purpose
Define hard rules for sponsor-level structural signaling.

## Scope
- Structural signaling only
- No marketing posture
- No mandate solicitation language

## Sections
1. Core intent
2. Hard rules (Rule A / Rule B)
3. Allowed vs disallowed patterns
4. Weak vs structural examples
5. Approval gate criteria

## Core intent
Sponsor-level presence is structural signaling. It is passive gravity, not outreach noise.

## Hard rules

### Rule A — No Future-Tense Authority Claims
Beacon copy must not imply authority-insert language such as:
- "we help"
- "we support"
- "we guide"
- "we solve"
- "we can"
- "we will"

Beacon copy can only describe structural reality. Do not place USG inside the sentence as actor.

### Rule B — No Embedded Mandate Offer
Beacon copy must not include mandate-offer or CTA language such as:
- "if this is you, let's talk"
- "reach out"
- "DM me"
- "book time"
- "we're working on X"
- "contact us"

Discovery must remain friction-driven and sponsor-initiated.

## Allowed vs disallowed patterns

### Allowed
- Problem-state architecture framing
- FID boundary definition failures
- Capital stack sequencing ambiguity
- Governance jurisdiction conflicts
- Structural risk allocation logic

### Disallowed
- Capability marketing
- Service positioning
- Generic inspirational language
- Sector cheerleading without structural claim
- Any call-to-action

## Weak vs structural examples

### Weak (reject)
- "Infrastructure in Africa needs better coordination."
- "We help sponsors unlock growth."
- "If this resonates, reach out."
- "Innovation will transform the region."

### Structural (approve candidate)
- "If FID requires five signatures and downside absorption is undefined, the platform is structurally unsignable."
- "Capital does not hesitate; it waits for sequencing clarity between governance authority and commitment boundary."
- "When capital stack seniority is unresolved at pre-FID, approval latency is a structural output, not an execution failure."

## Approval gate criteria
A beacon is publishable only if all are true:
1. Contains a concrete structural friction statement.
2. Uses commitment-boundary or FID language explicitly.
3. Contains zero Rule A phrases.
4. Contains zero Rule B phrases.
5. Contains no marketing/identity pitch language.
6. Is concise (target 60–140 words) and high density.
7. Maps to one live signal and one initiative context.

If any criterion fails, status = REJECT.
